## 4-Quadrant 매트릭스 개요

allganize `RAG-Evaluation-Dataset-KO` 300 Q&A 기준, 동일 retrieval(`gemma-embed-300m` + FAISS top-5) 위에 LLM 답변자 46종과 LLM judge 17종을 cross-judge.

```
              Local judge (8)           API judge (9)
            ┌─────────────────────────┬─────────────────────────┐
Local-gen  │ Q1                      │ Q3                      │
(12 LLM)   │  8 × 12 × 4 × 300       │  9 × 12 × 4 × 300       │
            │  = 115,200 calls        │  = 129,600 calls ✅     │
            ├─────────────────────────┼─────────────────────────┤
API-gen    │ Q2                      │ Q4                      │
(34 LLM)   │  8 × 34 × 4 × 300       │  8 × 34 × 4 × 300       │
            │  = 326,400 calls        │  = 326,400 calls ✅     │
            └─────────────────────────┴─────────────────────────┘
```

**Q3 + Q4 100% 완료** (456,000 calls). Q1 / Q2 일부 진행 중.

## RRF (Reciprocal Rank Fusion) 통합 방식

여러 judge 의 ranking 을 **합의 점수**로 변환:

`RRF_score(c) = Σ 1 / (k + rank_j(c))`

- `k = 60` (관례적 상수)
- `rank_j(c)` : judge `j` 의 leaderboard 에서 candidate `c` 의 순위
- 단순 평균보다 **outlier judge 의 영향이 작아짐** — 한 judge 가 이상치를 줘도 다른 judge 들이 일관되면 ranking 안정

각 cell `#R (acc)` = 해당 judge 에서 cand 의 rank + accuracy. 마지막 열 `RRF Rank (score)` = 모든 judge 통합 ranking + RRF 점수.

**Q3 12 cand × 9 judge**: top 5 ranking 이 모든 judge 에서 일치. 다른 judge 를 써도 같은 결과.

## 방법론

### 채점 프로토콜 (allganize 기반)
- 4 metric: `similarity / correctness / completeness / faithfulness`
- threshold = 4 (1-5 scale 에서 4 이상 = "좋음")
- majority vote: 4 metric 중 ≥2개가 ≥4 → "O" (정답 처리)
- accuracy = O / total Q (300)

### Reasoning / thinking 통일
- 로컬 judge: `JUDGE_MODE=nothink` (`enable_thinking=False`)
- OpenAI judge: `reasoning.effort='none'`, `max_output_tokens=64`
- Anthropic judge: thinking 비활성화 (default off)
- Gemini judge: OpenRouter `reasoning.effort='low'` (Pro/Flash thinking-class 모델)

### Fallback / 재처리
- **Anthropic Opus 4.7 → 4.6 fallback**: Q3 52건 + Q4 ~128건. Opus 4.7 이 `q142`/`q258` 같은 의료 prompt 를 `stop_reason: refusal` 로 거부 → 11가지 우회 모두 실패 → Opus 4.6 으로 재호출.
- **Sonnet 4.6 retry**: Q3 147건 + Q4 343건. main batch 의 `max_tokens=64` 가 작아 분석 텍스트만 출력 → `max_tokens=1024` 로 재호출.
- **Empty cand 보강**: Q4 16 cand-q pair (kimi-k2.6 q150 등) 의 첫 entry 가 빈 답변. 후속 retry entry 선택 → 8 judge × 4 metric = 512 calls 재평가.
- **gpt-5.4-pro q181/q223**: cand 파일에 누락 (298/300). Responses API 로 직접 호출 (reasoning=medium) → 8 judge 평가.

## 결론

LLM-as-Judge 매트릭스에서 **judge 사이즈·가족·reasoning 여부보다 채점 프로토콜이 ranking 을 결정**한다. RAG 답변 품질 측정에는:

1. 가장 비싼 reasoning judge (`opus-4-7`, `gpt-5.5`) 도 가성비 좋은 small judge (`gpt-5.4-nano`, `gemini-3-flash-lite`) 도 **동일한 RRF ranking** 을 제공.
2. Self-judge bias 는 1-2%p 수준으로 ranking 에 영향 없음.
3. 실용 관점: small / cheap judge 로 충분.
4. RAG 답변자 선택이 ranking 의 본질적 결정 요인.
